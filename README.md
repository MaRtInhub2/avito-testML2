1. **Feature engineering**  
   Я построил простые, но сильные признаки:
   - исторический поведенческий сигнал (`item_query_click_conv`)
   - совпадение категории запроса и объявления
   - совпадение локации
   - логарифм цены (более устойчив к выбросам)
2. **ML-модель**  
   В качестве модели я использовал логистическую регрессию:
   - это классический ML-алгоритм
   - он стабилен и интерпретируем
   - не переобучается на шумных данных
   - полностью соответствует требованиям задания
3. Ранжирование
   Предсказанную моделью вероятность контакта я использую как скор релевантности, по которому сортирую объявления внутри каждого `query_id`.
4. Формирование submission 
   Итоговый CSV содержит только `query_id` и `item_id`, отсортированные в порядке убывания релевантности.
